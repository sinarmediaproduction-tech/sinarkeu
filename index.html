📖 Panduan Lengkap Sinarkeu

Selamat Datang di Sinarkeu v8

Sinarkeu adalah aplikasi pencatatan keuangan berbasis web yang berjalan sepenuhnya di browser Anda. Data disimpan secara aman di cloud menggunakan Supabase dan terenkripsi dengan password pribadi Anda.

💡 Sinarkeu berjalan 100% di browser — tidak perlu install apapun. Cukup buka file index.html di browser.

☁️ Setup Akun & Project Supabase

Supabase adalah platform database PostgreSQL open-source yang menyediakan tier gratis. Sinarkeu menggunakannya sebagai backend penyimpanan data cloud.

Langkah 1 — Buat Akun Supabase
1
Buka supabase.com
Kunjungi https://supabase.com dan klik tombol "Start your project" atau "Sign Up".
2
Daftar dengan GitHub atau email
Disarankan menggunakan akun GitHub untuk kemudahan login. Akun gratis sudah cukup untuk penggunaan pribadi.
Langkah 2 — Buat Project Baru
1
Klik "New Project"
Dari dashboard Supabase, klik tombol New Project.
2
Isi detail project
Name: bebas, misal sinarkeu-kas
Database Password: buat password kuat — simpan baik-baik!
Region: pilih Southeast Asia (Singapore) untuk koneksi lebih cepat
Plan: Free sudah cukup
3
Tunggu project selesai dibuat
Proses pembuatan membutuhkan 1–2 menit. Tunggu hingga status berubah dari "Setting up" menjadi hijau.
Langkah 3 — Ambil URL & Anon Key
1
Buka Project Settings → API
Di sidebar kiri Supabase, klik ikon ⚙️ Settings, lalu pilih API.
2
Salin Project URL dan anon/public key
Project URL — bentuknya: https://xxxxxxxx.supabase.co
Project API Keys → anon / public — dimulai dengan eyJhbGciOiJIUzI1NiIs...

Kedua nilai ini yang dimasukkan ke form setup Sinarkeu.
⚠️ Jangan gunakan service_role key — selalu gunakan anon/public key saja untuk keamanan.
✅ Tier gratis Supabase mendapat 500 MB storage database, 2 GB bandwidth/bulan, dan 50.000 baris — lebih dari cukup untuk pemakaian pribadi bertahun-tahun.
🛢️ Membuat Tabel Database (SQL)

Sinarkeu membutuhkan 4 tabel di Supabase. Jalankan SQL berikut di SQL Editor Supabase (klik ikon Terminal/SQL di sidebar kiri).

⚠️ Jalankan semua SQL di bawah ini sebelum menghubungkan aplikasi. Cukup sekali saja.
Tabel 1 — transactions

Menyimpan semua data transaksi pemasukan dan pengeluaran.

SQL — Jalankan di Supabase SQL Editor
CREATE TABLE IF NOT EXISTS transactions ( id TEXT PRIMARY KEY, book_id TEXT NOT NULL DEFAULT 'b_default', device_id TEXT, type TEXT NOT NULL CHECK (type IN ('income','expense')), amount NUMERIC(15,2) NOT NULL DEFAULT 0, category TEXT, description TEXT, date TIMESTAMPTZ, attachment TEXT, created_at TIMESTAMPTZ DEFAULT NOW() ); -- Index untuk mempercepat query per buku CREATE INDEX IF NOT EXISTS idx_transactions_book ON transactions(book_id, date DESC); -- Aktifkan Row Level Security (RLS) — izinkan akses publik via anon key ALTER TABLE transactions ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow all for anon" ON transactions FOR ALL USING (true) WITH CHECK (true); -- Aktifkan upsert (insert or update) ALTER TABLE transactions REPLICA IDENTITY DEFAULT;
Tabel 2 — audit_logs

Menyimpan log audit setiap aksi (tambah, ubah, hapus transaksi).

SQL
CREATE TABLE IF NOT EXISTS audit_logs ( id BIGSERIAL PRIMARY KEY, book_id TEXT NOT NULL DEFAULT 'b_default', device_id TEXT, action TEXT, details TEXT, timestamp TIMESTAMPTZ DEFAULT NOW() ); CREATE INDEX IF NOT EXISTS idx_audit_book ON audit_logs(book_id, timestamp DESC); ALTER TABLE audit_logs ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow all for anon" ON audit_logs FOR ALL USING (true) WITH CHECK (true);
Tabel 3 — settings

Menyimpan setelan aplikasi: daftar buku, anggaran bulanan, dan konfigurasi Telegram.

SQL
CREATE TABLE IF NOT EXISTS settings ( id BIGSERIAL PRIMARY KEY, book_id TEXT NOT NULL, key TEXT NOT NULL, value TEXT, updated_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(book_id, key) ); CREATE INDEX IF NOT EXISTS idx_settings_book ON settings(book_id, key); ALTER TABLE settings ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow all for anon" ON settings FOR ALL USING (true) WITH CHECK (true);
Tabel 4 — backups

Menyimpan cadangan data harian secara otomatis.

SQL
CREATE TABLE IF NOT EXISTS backups ( id BIGSERIAL PRIMARY KEY, book_id TEXT NOT NULL, device_id TEXT, label TEXT, data TEXT, created_at TIMESTAMPTZ DEFAULT NOW() ); CREATE INDEX IF NOT EXISTS idx_backups_book ON backups(book_id, created_at DESC); ALTER TABLE backups ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow all for anon" ON backups FOR ALL USING (true) WITH CHECK (true);
✅ Cara menjalankan SQL di Supabase:
1. Login ke Supabase → buka project Anda
2. Klik ikon SQL Editor di sidebar kiri (ikon terminal / database)
3. Klik "New query"
4. Paste salah satu SQL di atas → klik "Run" (Ctrl+Enter)
5. Ulangi untuk semua 4 tabel
Verifikasi Tabel

Setelah menjalankan semua SQL, verifikasi tabel berhasil dibuat dengan query ini:

SQL — Cek tabel
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public' ORDER BY table_name;

Hasilnya harus menampilkan: audit_logs, backups, settings, transactions.

Opsi: Aktifkan Realtime (opsional)

Jika ingin melihat perubahan data real-time di multiple perangkat:

SQL — Aktifkan Realtime
ALTER PUBLICATION supabase_realtime ADD TABLE transactions; ALTER PUBLICATION supabase_realtime ADD TABLE audit_logs;

Buka Telegram → cari @BotFather → klik Start → ketik /newbot
Ikuti instruksi: beri nama bot dan username (harus diakhiri bot, misal: KasKuBot)
Salin Bot Token yang diberikan — format: 123456789:AAF-xxx...
2
Start bot Anda
Cari username bot yang baru dibuat di Telegram → klik Start.
Langkah ini wajib — bot tidak bisa mengirim pesan ke chat yang belum pernah di-Start.
3
Dapatkan Chat ID Anda
Buka @userinfobot di Telegram → klik Start → kirim pesan apapun.
Bot akan membalas dengan info akun Anda. Salin angka di bagian Id.
Untuk grup: tambahkan bot ke grup, lalu kirim pesan di grup → cek https://api.telegram.org/bot<TOKEN>/getUpdates

❓ Pertanyaan Umum (FAQ)
📱 Device Baru & Multi-Perangkat
?
Di device baru, apakah saya harus memasukkan password lama?
Tidak. Di device baru, localStorage kosong sehingga aplikasi membuka form Setup Pertama Kali, bukan layar login. Anda cukup isi Supabase URL + Anon Key lalu buat password baru (bebas, boleh sama atau berbeda). Data dari Supabase langsung ditarik otomatis setelah terhubung.

Yang WAJIB disimpan baik-baik: Supabase URL dan Anon Key — tanpa keduanya, Anda tidak bisa konek ke data di cloud.
?
Saya ganti HP / browser — apa yang perlu disiapkan?
Siapkan 2 hal: (1) File index.html — pindahkan via Google Drive, WhatsApp, email, dll. (2) Supabase URL & Anon Key — buka di Supabase dashboard → Settings → API. Password lama tidak diperlukan.
?
Apakah bisa buka di banyak device sekaligus?
Ya. Setiap device setup sendiri-sendiri dengan URL + Key yang sama. Setiap device bisa punya password berbeda — ini tidak masalah karena password hanya mengenkripsi URL & Key secara lokal, bukan mengunci akses ke Supabase. Data akan sinkron otomatis antar device.
🔐 Keamanan & Password
?
Saya lupa password, bagaimana cara memulihkan data?
Sayangnya password tidak dapat dipulihkan karena digunakan untuk enkripsi AES-256. Jika lupa, klik "Reset App" (akan menghapus semua data lokal) dan setup ulang. Data di Supabase masih ada — namun tidak bisa diakses tanpa password yang benar.
?
Apakah data saya aman?
URL dan Anon Key Supabase dienkripsi di browser menggunakan AES-256-GCM dengan key turunan dari password Anda. Data transaksi di Supabase diproteksi Row Level Security (RLS). Namun pastikan Anda tidak membagikan URL + Key + Password ke orang lain.
☁️ Koneksi & Sinkronisasi
?
Status cloud menunjukkan "CLOUD OFF" padahal sudah dikonfigurasi
Cek koneksi internet Anda. Jika internet OK, buka Kelola → Supabase dan klik ulang "Hubungkan & Sinkronkan". Pastikan URL dan Key masih valid di Supabase dashboard.
?
Data tidak tersinkron ke perangkat lain
Pastikan kedua perangkat menggunakan URL, Key, dan Password yang sama. Coba klik refresh halaman di perangkat yang datanya belum sinkron.
📊 Laporan & Export
?
Bagaimana cara backup data?
Klik tombol 🔒 Backup di header. Tersedia: backup lokal (browser), backup cloud (Supabase), dan export CSV/JSON. Backup cloud berjalan otomatis harian.
?
Bagaimana cara memindahkan data ke perangkat baru?
Cukup buka index.html di perangkat baru dan setup dengan URL, Key, dan Password yang sama. Data otomatis tertarik dari cloud. Atau gunakan Export JSON → Import JSON untuk transfer manual.
📲 Telegram
?
Notifikasi Telegram tidak terkirim
Pastikan: (1) Anda sudah klik Start di bot Telegram, (2) Bot Token dan Chat ID benar, (3) Bot tidak diblokir. Gunakan tombol "📤 Tes Kirim" untuk diagnosa — pesan error akan ditampilkan.
?
Bisa kirim notifikasi ke grup Telegram?
Bisa! Tambahkan bot ke grup Telegram, lalu gunakan Chat ID grup (dimulai dengan tanda minus, contoh: -100123456789) sebagai Chat ID di pengaturan.
